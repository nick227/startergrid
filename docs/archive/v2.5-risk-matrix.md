# V2.5 Platform Profile Risk Matrix

V2.5 keeps the POC narrow: fake onboarding, fake posting/submission, platform-profile schema confidence, and controlled-bubble receipts.

## New command

```bash
npm run poc:risk
```

Alias:

```bash
npm run poc:strict
```

## What it proves

- Baseline data still reaches GREEN.
- Broken dealership profiles become RED.
- Broken vehicle payloads become RED.
- Stale platform profiles become YELLOW or RED.
- Strict profile mode keeps medium-confidence / assisted marketplace paths from looking production-ready.

## Scenario set

| Scenario | Expected result | Purpose |
|---|---|---|
| BASELINE | GREEN | Happy path still works. |
| STRICT_PROFILE | GREEN/YELLOW | Separates mock-valid from partner-confirmed. |
| NEGATIVE_DEALERSHIP | RED | Missing dealer license/contact/address fields fail. |
| NEGATIVE_VEHICLE | RED | Invalid VIN, missing fields, missing media fail. |
| STALE_PROFILE | YELLOW/RED | Stale or review-needed platform profiles stop green. |

## Interpretation

`npm run poc:green` is the smoke test.

`npm run poc:risk` is the real confidence test.

A platform should only be considered launch-ready when it passes the baseline and has no strict-profile warnings that matter for the intended submission mode.
